# WindyAgent 知识库

WindyAgent 的知识分**两层**，AI 回答服务器相关问题（规则、商品、玩法、常见问答、插件用法等）时，
两层会**一起检索**——它先查这里，而不是凭空编造。

## 一、你的知识库（可编辑）

插件数据目录下的 `knowledge/` 文件夹就是**你自己的**知识库。**每个 `.md` 文件 = 一条知识，子目录 = 分类**。
把你服务器的规则、商品价格、玩法、常见问答写进来，AI 就懂你的服。两种编辑方式，内容互通：

1. **网页面板**（推荐日常）：控制台「知识库」页——目录树、Monaco 编辑器、Markdown 预览、AI 起草，
   支持 `[[双链]]`、反向链接、关系图。
2. **Obsidian**（推荐大批量整理）：`knowledge/` 本身就是一个标准 **Obsidian vault**，
   用 Obsidian「打开文件夹作为库」直接指向它即可，两边改的是同一批 `.md`，天然同步。
   （Obsidian 需你自己安装，App 免费；我们只是把这个文件夹做成了它能打开的库。）

首次启动会在这里放几份**填空模板**（`规则/ FAQ/ 商品/ 活动/`），照着填即可；删掉不影响插件运行。

## 二、内置参考库（只读，随插件更新）

插件还**内置**了一批官方文档（如 [[WindyAgent能力清单|CMI]]、DecentHolograms 等插件的 wiki 全文），
打进插件包里、**只读**。它们：

- **不出现在你的 `knowledge/` 目录树里**（不污染你的分类），只在 AI 检索时参与；
- **随插件版本自动更新**——你换新版插件、重启服务器，这批文档就是最新的，**无需你手动维护**；
- 所以 AI 被问到「CMI 怎么设置礼包冷却」「DecentHolograms 某命令什么权限」时，能直接引用官方文档作答。

要开关或挑选内置文档，见配置文件 `knowledge.reference` 段。

## 三、双链（wikilink）

正文里写 `[[条目标题]]` 链到另一条知识，`[[条目标题|显示文字]]` 可自定义显示文字。
面板和 Obsidian 都会渲染成可点击链接，并据此生成反向链接与关系图。

## 给管理员的建议

把「会反复被玩家问」「有明确规则/数值」的东西写进**你的知识库**最划算（价格、权益、玩法、处罚标准、
常见故障处理）。插件用法这类通用文档，内置参考库已经覆盖，不用你重复抄。相关能力见 [[WindyAgent能力清单]]。
