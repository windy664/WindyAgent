# 知识库（Obsidian vault）操作流程

这是一个**纯文字技能**：没有脚本，正文是给你（Agent）的操作规范。WindyAgent 的知识库 `knowledge/`
就是一个标准 **Obsidian vault**——每个 `.md` = 一条知识，子目录 = 分类。你**只用现有工具**操作它，
不要自己写代码、也不要去读写 vault 以外的任意文件。

## 读 / 搜
- 需要服务器特定信息（规则、价格、会员权益、玩法、常见问答等）时，**先用 `knowledge_search`** 检索，
  依据检索结果作答，不要凭空编造。搜不到就如实说「知识库里暂无」。

## 建 / 改
- 用 `knowledge_write` 新增或更新条目。只写**确有长期复用价值**的内容，别记一次性琐碎。
- 归类：用 `folder` 参数放进对应子目录（如 `规则`、`商品`、`活动/2026`）。常见分类见 [[插件说明·如何使用知识库]]。
- 更新已有条目：传它的 `id`（vault 相对路径，如 `规则/pvp规则`）即覆盖更新；改 `folder` 会把它移动到新分类。
- **双链**：正文里用 `[[另一条标题]]` 引用相关条目，`[[标题|显示文字]]` 可自定义显示文字。多用双链，
  知识之间连成网，面板的反向链接与关系图会据此自动生成。

## 纪律
- 写库属于管理操作：**仅可信来源（管理员 / 定时任务）可写**，普通玩家聊天触发的写入会被系统拒绝，别硬试。
- 沉淀运营记录、规则约定、FAQ 时，标题取得能被将来检索命中；正文用 Markdown，结构清晰。
- 不确定归到哪个分类时，宁可放根目录，别新建一堆零散目录。
