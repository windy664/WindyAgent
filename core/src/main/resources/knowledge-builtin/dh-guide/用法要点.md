# 全息（DecentHolograms）用法要点

「全息 / 悬浮字」= 悬浮在世界某坐标上的**持久**文字，跨重启留存，常用于出生点欢迎语、商店牌、规则告示、装饰。第一方精简说明；官方命令的确切用法可用 `describe_command` 现场探查 `dh`。

## 让 AI 代劳（首选）
直接说需求，如「在我脚下立个【欢迎】的悬浮字」「把出生点那块告示删了」。AI 会调用内置 **`hologram` 技能**：
- `action=create`：给 `text`（多行用 `|` 分隔）、`player`（按其位置建，落在头顶上方）或 `x/y/z(+world)` 坐标；省略 `id` 自动生成。
- `action=delete`：给 `id` 删除。
- **改内容/移动**：技能不单独支持编辑 → 先 `delete` 同 id 再 `create`。
- 坐标只有子服有（Velocity 中心端没有），技能须经 `run_skill_on_server` 发到玩家所在子服；内核已自动注入请求者所在子服。

## 手动命令（服主自己敲）
主命令 `/dh`（也可能是 `/holograms`）。常见子命令：`create <名> [文本]`、`addline <名> <文本>`、`setline <名> <行号> <文本>`、`movehere <名>`（移到你脚下）、`delete <名>`、`list`。**确切子命令/参数以 `describe_command dh` 探到的为准**，不同版本略有差异。

## 内容能力
- **颜色/样式**：Minecraft `§` 颜色码与 `&` 码（`§c`红 `§b`天蓝 `§e`黄 `§7`灰 `§8`深灰 `§f`白；`§l`粗 `§o`斜 `§n`下划线 `§m`删除线 `§k`乱码）。
- **占位符**：装了 PlaceholderAPI 时支持 `%player_name%`、`%server_online%` 等动态占位（每隔一段自动刷新）。
- **多行**：一个全息可多行，逐行从上往下排；行距、可见距离等在 DH 配置里调。
- **图标行**：`#ICON:` 前缀可放物品图标（语法以本服 DH 版本为准，探查后再用）。

## 常见注意
- id 唯一：重名会冲突，本技能遇重名会拒绝覆盖。
- 建在实心方块里会被埋，按玩家建时已自动抬到头顶上方；用坐标时自己留出高度。
- 删除是永久的（持久全息会同步删存档），删前确认。
